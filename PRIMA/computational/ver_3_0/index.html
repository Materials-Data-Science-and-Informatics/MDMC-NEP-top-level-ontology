Description

This is the computational workflow module of the Provenance Information for Materials Science (PRIMA) Ontology, version 3.0, aligned with PMDco v3 and based on BFO (Basic Formal Ontology).

Model Preparation c

IRI https://purls.helmholtz-metadaten.de/prima/computational#ModelPreparation
Is Defined By https://purls.helmholtz-metadaten.de/skosmos/prima/modelpreparation
Description

Set of actions carried out by one or more Research Users, possibly using Research Software, to define and/or formulate a Model making use of Research Data (typically, but not limited to, Reference Data after Data Processing) as input for parametrization, boundary conditions, or validation. A Model Preparation may require the use of Equipment. The output of Model Preparation is Model.

Simulation c

IRI https://purls.helmholtz-metadaten.de/prima/computational#Simulation
Is Defined By https://purls.helmholtz-metadaten.de/skosmos/prima/simulation
Description

Computational Data Acquisition performed by one or more Research Users on a Model to manipulate its Settings using a specific computational and/or theoretical Technique in order to study, predict, or optimize the behaviour and performance of existing or proposed features and properties of a physical System that would otherwise be too complex, too large/small, too fast/slow, too dangerous, inaccessible, or unacceptable to engage and/or control. Examples of Simulation are: multiscale simulation, finite-element simulation, molecular dynamics simulation, discrete dislocation dynamics simulation.

Model c

IRI https://purls.helmholtz-metadaten.de/prima/computational#Model
Is Defined By https://purls.helmholtz-metadaten.de/skosmos/prima/model
Description

Digital representation of a System, primary output of any kind of Model Preparation, aimed to be used in one or more Calculations or Simulations for its description or for predictions of its behaviour. A Model represents the System by direct similitude (e.g., small-scale replica) or by capturing in a logical framework the relations between its properties (e.g., mathematical Model). A Model typically consists of Settings which may be stored in one or more files.
Sub Class Of core:System
Restriction core:hasSetting op some core:Settings c
